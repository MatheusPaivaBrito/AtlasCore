# Glossary

| Term | Meaning in AtlasCore |
| --- | --- |
| Product API | API that owns product-facing behavior. |
| Platform API | API that owns internal platform capability. |
| Worker | Non-HTTP process for background jobs. |
| Adapter | Infrastructure implementation detail around a tool/provider. |
| Capability | Coherent responsibility with its own language and workflows. |
| Bounded Context | Boundary where a model/language is consistent. |
| Outbox | Table/pattern used to publish events reliably after DB changes. |
| Event Sourcing | Modeling strategy where events are the source of truth. |
| Projection | Read model built from events. |
| Shared Kernel | Small package of cross-service primitives. |
